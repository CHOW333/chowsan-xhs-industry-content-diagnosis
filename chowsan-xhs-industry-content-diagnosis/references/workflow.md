# Workflow

## Input

Minimum input:

- Industry.

Useful optional fields:

- City or business district.
- Single store or chain.
- Main products or services.
- Main customer group.
- Current problem: exposure, consultation, store visits, repeat purchase, private community, or paid conversion.

## Diagnosis Chain

### 1. One-line judgment

Use:

```text
This industry on Xiaohongshu is not mainly about [surface action]. It is about [real user decision].
```

Examples:

- A convenience store is not a brand account. It is a nearby instant consumption decision guide.
- A community fruit store is not a price board. It is a daily fruit buying guide for nearby families.
- A moving service is not selling labor. It is reducing the user's fear that the moving process will get out of control.

### 2. Industry ecology

Classify the business:

- High-frequency instant consumption: convenience store, breakfast, tea drink, fruit, light meal.
- Low-frequency strong scene: photography, homestay, moving, car care, foot care.
- High-concern service: fitness, housekeeping, education, repair, home improvement.
- Emotional and ritual consumption: flowers, parent-child photos, restaurants, birthdays.
- Nearby daily solution: community shop, school-side shop, office-area shop.

### 3. User search intent

Always split into four layers:

- Local words: city, district, subway station, school, office building, community, street.
- Scene words: breakfast, after work, rainy season, first time, weekend, child, elderly, gift.
- Problem words: how to choose, avoid mistakes, worth it, how much, suitable, will I be pushed to buy.
- Category words: fruit store, car wash, gym, convenience store, housekeeping, photography.

### 4. What not to post

Name the common wrong content:

- Price list.
- Product display.
- Environment photos.
- "Very professional" slogans.
- Owner self-talk.
- Before-and-after comparison with no context.
- Daily updates without decision value.

Use this line:

```text
These are not forbidden, but they do not give the user an immediate reason to act.
```

### 5. What to post instead

Translate product into user scene:

```text
product/service + user scene + concrete problem + action reason
```

### 6. Content directions

Default seven directions:

| Type | Function |
| --- | --- |
| Search-question content | Capture active demand |
| Scene-trigger content | Make users remember when to use the business |
| Avoid-mistake content | Build trust |
| Price-combination content | Lower decision cost |
| Owner/staff perspective | Add reality and credibility |
| Local-position content | Capture nearby search |
| Case/feedback content | Push consultation or store visit |

### 7. Account positioning

Use:

```text
place + audience + scene + guide
```

Examples:

- Office-area convenience eating guide.
- Community fruit buying guide.
- Subway-station breakfast guide.
- Business-district car owner care guide.

### 8. Plan

Use the 7-day sequence:

1. Positioning.
2. High-frequency scene.
3. User concern.
4. Price or combination.
5. Avoid-mistake judgment.
6. Owner/staff perspective.
7. Case, feedback, ranking, or gift scene.

### 9. Bridge

Connect Xiaohongshu to:

- Store visit.
- Comment consultation.
- Private community.
- Reservation.
- Repeat purchase.
- Gift or package pre-order.

Do not force private traffic inside the note. Prefer: content builds trust, store or consultation completes the bridge.
